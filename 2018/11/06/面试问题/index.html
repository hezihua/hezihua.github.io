1.iframe有什么缺点？

iframe 会阻塞主页面的onload事件

iframe与主页面共享连接池，而浏览器对相同域的连接有限制，所以会印象页面的并行加载，使用iframe之前要考虑这两个缺点，如果需要使用iframe，最好是通过javascript 动态给iframe添加src属性值，这样可以绕开这两个问题。

2.使用代理模式实现图片的延迟加载

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="ie=edge">
    <title>Document</title>
    <style>

    div{color: #222722;}
    div:hover{color:red;}
    </style>
</head>
<body>
    <div>
        <span class="span">111</span>
    </div>
    <script>
        // myImage负责往页面中添加img标签
var myImage = (function(){
    var imgNode = document.createElement('img');
    imgNode.name = "img"
    document.body.appendChild(imgNode);
    return {
        setSrc: function(src) {
            imgNode.src = src;
        }
    }
})();   

//proxyImage负责预加载图片，并在预加载完成之后把请求交给本体myImage
var proxyImage = (function(){
    var img = new Image;
    img.onload = function() {
        myImage.setSrc(this.src);
    }
    return{
        setSrc: function src(src) {
            myImage.setSrc('assets/love.jpg');
            img.src = src;
        }
    }
})();
proxyImage.setSrc('assets/images.jpg');
    </script>
</body>
</html>

5.Doctype的作用，如何区分严格模式和混杂模式，他们有何意义

声明阅读程序应该用什么样的规则来解释文档中的标记

严格模式是根据web标准去解析，混杂模式是一种向后兼容的解析方法，就是出现了过去的版本的web标准也能解析

严格模式是现代浏览器标准显示方式，混杂模式防止老站点无法工作

6.如何理解vue的mvvm框架，它与jq的区别在哪里？

mvvm框架，全程model-view-viewmodel，其中视图模型是一个值转换器，从数据模式中公开数据对象，以便轻松管理和呈现数据对象，只需关心数据操作；jquery提供了大量操作dom的API,dom操作；
